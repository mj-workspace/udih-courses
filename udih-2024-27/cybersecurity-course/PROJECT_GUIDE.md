# PROJECT_GUIDE.md

## Какво е това

Съдържателно репозитори за **курс по киберсигурност и защита на данните**, предназначен за 25–30 служители на ВиК дружество (софтуерни инженери, IT специалисти, сисадмини). Организиращо тяло: Академия по национална и информационна сигурност. Формат: онлайн (Google Meet), 12 учебни часа в 2 дни.

Репозиторито съдържа:
- Изходни Markdown документи (план за курса, ръководство за лектора)
- **React уеб приложение** (`lecturer-guide/`) — интерактивен "пищов" за лектора, използван по време на преподаване
- Спецификации и документация (`.specs/`, `context/`)

## Курсова структура

| Модул | Тема | Продължителност |
|---|---|---:|
| Модул 1 | Въведение в киберсигурността (CIA Triad, заплахи, ВиК контекст) | 50 мин |
| Модул 2 | Фишинг, злонамерен софтуер и онлайн измами | 90 мін |
| Модул 3 | Сигурност на пароли и автентикация (2FA/MFA) | 50 мін |
| Модул 4 | Защита на данни (GDPR, NIS2, класификация) | 80 мін |
| Модул 5 | Мрежова сигурност и инфраструктура | 120 мін (разделен: 30 мін Ден 1 + 90 мін Ден 2) |
| Модул 6 | Политики, процедури и реакция при инциденти | 120 мін |

**Ден 1 (09:30–17:00):** Въведение → М1 → М2 → М3 → М4 → М5 Част 1 → Обобщение
**Ден 2 (09:30–17:00):** Преговор → М5 Част 2 → М6 → Финален изпит (120 мін)

**Изпити:** Входен (преди курса, без време) + Финален (120 мін) — 20 въпроса, 4 варианта, 1 верен. Google Forms. Нарочно лесни (държавно изискване за обучение).

**Ключово ограничение:** SCADA/ICS е изрично изключено (отделен курс).

## Lecturer-Guide приложение

### Стартиране

```bash
cd lecturer-guide && npm run dev
```

Отваря се на http://localhost:5173. Работи само локално.

### Архитектура

**Стек:** React 19 + Vite 8 + Tailwind CSS 4. Без router, без state management библиотека.

**Layout:** 40% sidebar (ляво) + 60% основна зона (дясно). Текст в основната зона: 17px.

**Sidebar** е групиран в три секции:
1. Без етикет: План на курса, Входен изпит
2. "Ден 1": Въведение, Модули 1–4, М5 Част 1, Обобщение
3. "Ден 2": Преговор, М5 Част 2, Модул 6, Финален изпит

Когато модул е активен, неговите подсекции (напр. 1.1, 1.2, 1.3, 1.4) се показват inline под него в sidebar-а.

**Навигация:** `navigationOrder` — плосък масив от 32 section ID-та в хронологичен ред. Бутоните Предишна/Следваща (долу вдясно) минават през тях последователно.

**Модул 5 split:** `module5.js` експортва един обект с 4 секции. `index.js` го разделя на `module5-part1` (секция 5.1) и `module5-part2` (5.2–5.4) чрез `.filter()` — един source of truth.

### Data модел

```
CourseModule {
  id: string              // "module1", "intro", "exam"
  title: string
  duration: string        // "50 мін"
  timeMarker: string      // "Ден 1, 09:40 – 10:30"
  note: string            // Бележка за лектора (italic под заглавието)
  sections: Section[]
}

Section {
  id: string              // "1.1", "intro-main", "exam-main"
  title: string
  talkingPoints: TalkingPoint[]
  examples: string[]      // Реални случаи, аналогии
  transition: string      // "ПРЕХОД →" текст
}

TalkingPoint {
  text: string            // Поддържа **bold** синтаксис
  children?: TalkingPoint[] // Разгъващи се подточки (отворени по подразбиране)
  detail?: string         // Подробно обяснение — отваря DetailPopup при клик
}
```

### Визуални типове съдържание

| Тип | Визуализация |
|---|---|
| talking-points | Списък с точки, разгъващи се подточки, dotted underline за detail popup |
| examples | Amber карта с 💡 заглавие |
| transition | Зелен блок с → икона и "ПРЕХОД" етикет |
| time-marker | Purple pill badge с ⏱ |

### Detail Popup система

Talking points с `detail` поле стават кликваеми:
- **Leaf item** (без children): текстът е с пунктирано подчертаване → клик отваря popup
- **Parent item** (с children): ℹ иконка вдясно → клик отваря popup

Popup-ът е модален, затваря се с ✕, Escape или клик извън. Поддържа `**bold**` и `\n\n` за параграфи.

## Как да добавяш/редактираш съдържание

### Добавяне на talking point с popup

В съответния data файл (напр. `module1.js`):
```js
{
  text: '**Термин:** кратко описание',
  detail: 'Подробно обяснение.\n\nВтори параграф с **bold** текст.'
}
```

### Добавяне на подточки

```js
{
  text: 'Заглавна точка:',
  children: [
    { text: '**Подточка 1:** описание' },
    { text: '**Подточка 2:** описание', detail: 'Подробно...' },
  ]
}
```

### Добавяне на нова секция към съществуващ модул

1. Добави `Section` обект в `sections[]` масива на модула в неговия data файл
2. В `navigation.js`: добави section ID в `navigationOrder` на правилната позиция
3. В `navigation.js`: добави mapping в `sectionToModule`

### Добавяне на нов модул

1. Създай нов data файл (напр. `newModule.js`)
2. Импортирай го в `index.js` и добави в `modules` обекта
3. В `navigation.js`: добави sidebar entry в `sidebarGroups`, section ID-та в `navigationOrder`, mappings в `sectionToModule`

## Езикова конвенция

Цялото съдържание е на **български**. Техническите термини използват английските си еквиваленти в скоби: "Поверителност (Confidentiality)", "Рансъмуер (Ransomware)", "Firewall (Защитна стена)".

## Ключови решения

- Приложението е **лично за лектора** — не е видимо за курсистите
- Съдържанието е **мигрирано 1:1** от `Ръководство_за_лектора.md` (852 реда, верифицирано)
- **Без автоматизирани тестове** — ръчна верификация, един потребител
- **Без dark mode, mobile, search** — out of scope
- Поддръжката се прави изцяло от **Claude Code** — редакции на data файлове
